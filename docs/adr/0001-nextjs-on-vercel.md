# Next.js on Vercel instead of the SPA + Fastify split

The v2 spec proposed a Vite SPA with a separate Fastify/NestJS server, but the app is hosted on Vercel, which doesn't run long-running servers — that stack would force a second host for the API alone. We chose a single Next.js (App Router) deployable: UI and the programmatic API (`/api/*` route handlers) ship together, with Postgres on Neon (Singapore), Drizzle ORM, and Better Auth (Lucia, named in the spec, was deprecated in 2025). Cost accepted: no long-running process, so websocket-based real-time is out — acceptable because real-time was explicitly a nice-to-have and Vercel cron covers scheduled needs. Storybook and XState were dropped as unjustified for a solo-built app whose only state machine is "all sub-tasks checked → milestone complete," computed server-side in one place.
